API RULES

Wes services format and common points

format?

Restful service (JSON In and Out), POST Request, Header : Content-Type: application/json.call sample

DATA?

All supported data, either input or output, are strings that meet the constraints requested and must be included in the json file (not null), even if empty

Header?

All messages (request or response) are preceded by a message identification (idmsg), chosen by the sender of the message (random or with an order), and returned by the recipient (same value), used to correlate responses with requests. With constraint number fixed length 12 not null.

Response?

All responses must contain a status object indicating the status of the response and the description of the error in case of error.

Keys?

sender key : 142DF54EF47ZE78B, recipient key : 552A15CF5889EAFF

mac value?

The mac value in exchanges represents a data hash encryption whose key is shared between SS and MB during the integration phase, ensuring the sender's authenticity. (could be ignored during the preparation phase). Mac Request is ( Mac = first 16 of SHA-256(idmsg+sender key)), Mac Response is ( Mac = first 16 of SHA-256(idmsg+recipient key)).mac sample
